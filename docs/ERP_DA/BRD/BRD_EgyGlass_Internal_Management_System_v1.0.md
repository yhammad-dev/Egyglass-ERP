## 1. الملخص التنفيذي

**EgyGlass** (تأسست 2001، القاهرة – مصر الجديدة) شركة متخصصة في الزجاج المعماري (الواجهات الزجاجية، Spider Systems، كبائن الدش، الكلادينج). تهدف إلى بناء نظام إدارة مؤسسي داخلي (ERP مخصص) يربط 6 إدارات رئيسية في سير عمل رقمي واحد يتتبع المشروع من طلب العميل حتى التسليم النهائي.

**المشكلة الجوهرية:** العمل حاليًا يعتمد على جداول Excel منفصلة (شيت لكل عميل) وتواصل يدوي بين الإدارات بدون تتبع مركزي أو أرشيف تاريخي أو إشعارات آلية، مما يؤدي إلى أخطاء في المقاسات، فقدان معلومات، وتأخير في التسليم.

**النطاق العام:** تغطية دورة حياة المشروع الكاملة — استقبال طلب العميل، متابعة، معاينة، تسعير، تعاقد، تصنيع، تركيب، ومحاسبة.

## 2. نطاق المشروع والحدود

### داخل النطاق

| الوحدة الوظيفية | الوصف |
|----------------|-------|
| إدارة التسويق والمبيعات | لوحة مدير لتوزيع الطلبات، شاشات عملاء بمستويات صلاحية، تحديثات مؤرخة، بروفايل عميل كامل، إدارة "الظَن" والعملاء المحتملين، أسباب الرفض |
| إدارة المعاينات | استلام طلبات المعاينة، جدولة، رفع مقاسات، إرسال للمكتب الفني، تسليم رسم تنفيذي + بطاقة إكسسوار |
| المكتب الفني | إعداد عروض الأسعار، إعادة تسعير، رسوم PDF للمصانع، إضافة بنود خاصة |
| المشتريات والمخازن | أوردر تصنيع مُوَحَّد، مراجعة مطابقة، إرسال للمصانع، تتبع تاريخ التصنيع، إشعار الجاهزية، تكاليف المشروع الكاملة |
| التركيبات | جدولة تركيب، تسجيل بنود خاصة (بدل كسر، خطأ تصنيع، خطأ مكتب فني)، صور وتقارير بعد التسليم |
| الحسابات | فواتير إلكترونية، مستخلصات، سجل شامل لكل عميل |

### خارج النطاق (صراحة أو ضمنيًا)

- الموارد البشرية والرواتب
- إدارة الأصول والمعدات
- بوابة عملاء خارجية (Customer Portal)
- التكامل مع أنظمة التسويق الرقمي
- إدارة المصنع/المصانع (الإنتاج الداخلي)
- تكامل واتساب/بريد إلكتروني آلي

## 3. الأهداف التجارية ومعايير النجاح

### الهدف الرئيسي
تحويل العمليات اليدوية المتفرقة إلى سير عمل رقمي موحد يقلل الأخطاء بين الإدارات ويوفر رؤية شاملة لحالة كل مشروع.

### الأهداف الثانوية
1. بروفايل عميل كامل مع هستوري من أول تواصل حتى تسليم
2. إشعارات آلية بين الإدارات عند انتقال المشروع لمرحلة تالية
3. تتبع التكاليف الكاملة لكل مشروع
4. تمكين تغطية الموظفين الغائبين بدون فقدان معلومات

### مؤشرات الأداء الرئيسية (KPIs)

| المؤشر | المستهدف |
|--------|----------|
| متوسط وقت دورة المشروع (طلب → تركيب) | تقليل ≥ 30% |
| نسبة الأخطاء بين الإدارات (مقاسات غير مطابقة) | تقليل ≥ 80% |
| وقت استرجاع معلومات عميل | < 30 ثانية |
| دقة تتبع التكاليف | ≥ 95% |

## 5. المتطلبات غير الوظيفية

| الفئة | المتطلب | التفاصيل |
|-------|---------|----------|
| الأمان | صلاحيات مبنية على الأدوار (RBAC) | مدير ← كل الصلاحيات. موظف ← عملاءه + بحث في زملائه |
| التدقيق | Audit Trail كامل | كل إجراء مُؤرَّخ ومُسجَّل باسم المستخدم |
| التوافر | واجهة Responsive | للاستخدام من الموبايل (خاصة التركيبات والمعاينات الميدانية) |
| النسخ الاحتياطي | يومي تلقائي | حماية جميع البيانات المالية والمشاريع |
| الأداء | استجابة < 2 ثانية | للشاشات الأساسية |
| التكامل | API مفتوح | للفاتورة الإلكترونية وأي تكاملات مستقبلية |
| اللغة | واجهة عربية كاملة (والإنجليزية اختياري) | اللغة الأساسية للمستخدمين هي العربية |
| النظام الأساسي | Web-based (سحابي أو On-Premise) | يُحدّد بعد تقييم الميزانية |

## 8. الخطوات التالية

### الأسبوع 1-2
- اجتماع عاجل مع محمد فاروق (المكتب الفني) لاستخلاص المتطلبات
- اجتماع مع الإدارة العليا لتحديد: الميزانية، الجدول، الأولويات
- توضيح متطلبات الفاتورة الإلكترونية (ETA)
- جرد الأنظمة الحالية في الشركة
- تحديد عدد المستخدمين النهائي والأدوار

### الأسبوع 3-4
- إنتاج وثيقة PRD كاملة (Product Requirements Document)
- تصميم Wireframes و User Flows
- تحديد الـ Tech Stack النهائي

### الأسبوع 5-8
- بناء MVP يغطي: المبيعات → المعاينات → المكتب الفني
- اختبار MVP مع فريق المستخدمين
- خطة تدريب وتطوير
