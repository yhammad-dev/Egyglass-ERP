# تحليل مشروع EgyGlass وتحويله لنظام متكامل

## أولاً: هيكل الشركة والأقسام

```
المبيعات والتسويق (طارق)
       │
       ▼
    المعاينات (حسن بهاء)
       │
       ▼
  المكتب الفني (م. محمد فاروق)
       │
       ▼
 المشتريات والمخازن (شكري)
       │
       ▼
    التركيبات (إسلام السيد)
       │
       ▼
   الحسابات (راندا)
```

---

## ثانياً: تحليل سير العمل الحالي (As-Is Workflow)

### المرحلة 1: المبيعات والتسويق
- استقبال طلب العميل
- إرسال عرض سعر (عن طريق المكتب الفني)
- متابعة العملاء وتقسيمهم إلى مراحل:
  - **ظن** (تم إرسال عرض السعر)
  - **تحت الظن** (قيد المتابعة)
  - **عينات** (يحتاج عينات)
  - **معينات** (يحتاج مساعدة)
- عند الرفض → تسجيل سبب الرفض
- عند الموافقة → تعاقد → تحويل للحسابات والتنفيذ

#### مشاكل المرحلة 1:
- لا يوجد نظام لتوزيع العملاء على مندوبي المبيعات
- لا يوجد هيكل لتغطية الموظف الغائب
- لا يوجد نظام لتسجيل المتابعات والأبديتس
- العميل مش عنده بروفايل موحد بالتاريخ الكامل

### المرحلة 2: المعاينات
- استقبال طلب معاينة من التسويق أو المشروعات
- البيانات المطلوبة: عرض سعر، عنوان، رقم تواصل، location
- ترتيب معاينة مع العميل
- إخراج المقاسات
- إرسال المقاسات للمكتب الفني (للتسعير أو إعادة التسعير)
- بعد التعاقد: استلام أمر تصنيع → مخرجان:
  - رسمة تنفيذية ← المهندس محمد فاروق (المكتب الفني)
  - بطاقة إكسسوار ← شكري (المخازن)

#### مشاكل المرحلة 2:
- لا يوجد توحيد للبيانات المرسلة
- لا يوجد tracking لحالة المعاينة

### المرحلة 3: المكتب الفني
- تسعير أولي لعروض الأسعار
- استلام مقاسات المعاينة
- رسم PDF تنفيذي (يتجه للمصانع)
- إضافة بنود إضافية (صنفرة، تعديلات)

#### مشاكل المرحلة 3:
- لا يوجد نظام لمراجعة الرسومات قبل التصنيع
- لا يوجد ربط بين الرسم التنفيذي وبطاقة الإكسسوار

### المرحلة 4: المشتريات والمخازن
- استلام أمر تصنيع (رسم معاينات + رسم مهندس + بطاقة إكسسوار)
- مراجعة كل المكونات (مطلوب قسم مراجعة مستقل)
- إرسال PDF للمصانع
- تجهيز بطاقة الإكسسوار
- إضافة بنود إضافية (صنفرة، بدل كسر، خطأ تصنيع، خطأ مكتب فني)
- متابعة التصنيع والتواريخ المتوقعة

#### مشاكل المرحلة 4:
- **لا يوجد قسم مراجعة** قبل وصول الأمر للمخازن
- لا يوجد tracking للشحنات وحالة التصنيع
- لا يوجد نظام لإدارة بنود إضافية

### المرحلة 5: التركيبات
- استلام notification من المخازن (المشتريات خلصت)
- ترتيب معاد التركيب مع العميل
- تسجيل تاريخ التركيب
- إضافة بنود بعد التركيب:
  - بدل كسر
  - خطأ تصنيع
  - خطأ مكتب فني
  - تأجيل العميل
- رفع صور وتقارير بعد التركيب

#### مشاكل المرحلة 5:
- لا يوجد نظام لتوزيع فِرق التركيب (على أساس location والطرق)
- لا يوجد توثيق للبنود الإضافية بعد التركيب
- لا يوجد app للفنين في الموقع

### المرحلة 6: الحسابات
- استلام الكوتيشن بعد التعاقد
- تحصيل الدفعة المقدمة
- تسجيل لكل عميل في شيت منفصل
- إصدار فواتير إلكترونية
- إصدار مستخلصات
- شيت نهائي بكل المبالغ المستحقة

#### مشاكل المرحلة 6:
- **Excel sheets منفصلة لكل عميل** (كارثة محاسبية)
- لا يوجد تكامل مع باقي الأقسام
- لا يوجد نظام للمستخلصات والفواتير الإلكترونية

## ثالثاً: النظام المقترح (To-Be System)

### النظام المقترح: **ERP متكامل لإدارة شركة زجاج**

### الوحدات المقترحة:

#### 1. Module: CRM ونظام المبيعات
- **إدارة العملاء**: بروفايل موحد لكل عميل بالتاريخ الكامل
- **توزيع العملاء**: المدير يوزع العملاء على المندوبين
- **تغطية الغياب**: المندوب يقدر يشوف عملاء زميله ويبعت لهم
- **مراحل البيع**: automatic tracking للمراحل
- **الأبديتس**: تسجيل متابعات يومية بالوقت والتاريخ
- **تقارير**: تقارير مبيعات، أسباب الرفض، أداء المندوبين

#### 2. Module: إدارة المعاينات
- **طلب معاينة**: Form موحد (location, address, phone, attachments)
- **جدولة المعاينات**: تقويم للمعاينات
- **تسجيل المقاسات**: رفع المقاسات والصور
- **حالة المعاينة**: tracking من البداية للنهاية

#### 3. Module: المكتب الفني
- **عروض الأسعار**: إدارة عروض الأسعار
- **الرسومات**: ربط الرسومات التنفيذية بالأوردرات
- **التسعير**: تسعير أولي وتعديلات
- **البنود الإضافية**: إضافة بنود (صنفرة، تعديلات)

#### 4. Module: المشتريات والمخازن
- **أمر التصنيع**: نظام متكامل لإدارة أوامر التصنيع
- **المراجعة**: خطوة مراجعة قبل التصنيع
- **التصنيع**: tracking مع المصانع
- **المخازن**: إدارة بطاقات الإكسسوار والمخزون
- **بنود إضافية**: صنفرة، بدل كسر، خطأ تصنيع

#### 5. Module: التركيبات
- **توزيع الفرق**: على أساس location والطرق
- **جدولة التركيب**: تقويم للمواعيد
- **توثيق**: صور وتقارير بعد التركيب
- **بنود ما بعد التركيب**: تسجيل الملاحظات

#### 6. Module: الحسابات (Accounting)
- **الفواتير**: فواتير إلكترونية (ERP format)
- **المستخلصات**: إدارة مستخلصات العملاء
- **المدفوعات**: تحصيل وتتبع المدفوعات
- **التسوية**: شيت موحد للمبالغ
- **التقارير**: حسابات كل عميل

#### 7. Module: dashboard و التقارير
- لوحة تحكم للمديرين
- مؤشرات الأداء (KPI)
- تقارير المبيعات والمخازن والحسابات

## رابعاً: تكنولوجيا مقترحة

```yaml
نظام واجهة المستخدم (Frontend):
  - إطار العمل: React / Next.js أو Angular
  - واجهة المستخدم: Bootstrap أو Material UI (واجهات عربية)
  - Mobile App: Flutter أو React Native (للفنين في الموقع)

نظام الخادم (Backend):
  - API: Node.js (NestJS) أو .NET Core أو Django
  - Database: PostgreSQL (للمعاملات) + MongoDB (للمرفوعات والصور)
  - التخزين: سحابي أو محلي

نظام المحاسبة:
  - تكامل مع أنظمة الفواتير الإلكترونية (حسب بلدك: مصر/السعودية/الإمارات)
  - ERPNext (Open Source) أو Odoo (Open Source)

ميزات عامة:
  - النظام عربي بالكامل
  - صلاحيات وأدوار (Admin, مدير, مندوب, فني)
  - إشعارات (push notifications)
  - نظام مراقبة في الوقت الحقيقي
  - API للتكامل مع الأنظمة الأخرى
```
