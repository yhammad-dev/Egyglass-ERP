## الرؤية والمتطلبات

| الرؤية والمتطلبات - الإدارة العليا |  |  |  |  |  |
|---|---|---|---|---|---|
| EgyGlass - Top Management | جهة الاتصال: المالك / CEO |  |  |  |  |  |
| المعرف | المتطلب | شرح المقصود | مثال توضيحي | مخرجات النظام | الأولوية |
| CEO-R01 | تحديد رؤية النظام ونطاقه | توضيح ما إذا كان النظام المطلوب هو ERP متكامل يشمل جميع الإدارات أم حل بسيط لربط الإدارات الحالية فقط | مثال: هل نريد نظام يشمل المحاسبة والمخازن بالكامل أم فقط ربط المبيعات بالمعاينات والمكتب الفني؟ | وثيقة رؤية المشروع (Project Vision Document) تحدد النطاق والأهداف | فورية |
| CEO-R02 | تحديد الميزانية والموارد | تحديد المبلغ المالي المخصص للمشروع (برمجيات،استضافه، أجهزة، تدريب، دعم فني) | مثال: الميزانية تشمل التطوير والتدريب والدعم لمدة عام | ميزانية معتمدة وجدول زمني للصرف | فورية |
| CEO-R03 | تحديد الجدول الزمني والمراحل | وضع خطة زمنية واقعية للمشروع مع مراحل تسليم واضحة | مثال: المرحلة الأولى (CRM + معاينات) خلال 3 أشهر، المرحلة الثانية (مشتريات + تركيب) خلال شهرين | خطة المشروع (Project Plan) مع milestones وتواريخ | فورية |
| CEO-R04 | اختيار البنية التقنية | تحديد ما إذا كان النظام سحابي (Cloud) أم على سيرفر داخلي (On-Premise) | مثال: نظام سحابي على Azure/SaaS يقلل تكاليف البنية التحتية ويسهل الدعم | قرار تقني موثق مع مبررات الاختيار | عالية |
| CEO-R05 | تحديد هيكل الحوكمة | تحديد مسؤوليات اتخاذ القرار وصلاحيات المستخدمين داخل النظام | مثال: مدير النظام (Admin) له صلاحيات كاملة، مدراء الأقسام يشوفون تقارير أقسامهم فقط | نموذج الصلاحيات (RBAC) وهيكل الحوكمة | عالية |
| CEO-R06 | توفير لوحات مؤشرات الأداء | إنشاء Dashboards تظهر مؤشرات الأداء الرئيسية (KPIs) للمتابعة اليومية | مثال: لوحة المدير تظهر عدد المشاريع النشطة، المبيعات الشهرية، المشاريع المتأخرة | لوحة مؤشرات تفاعلية مع تقارير دورية | عالية |
| CEO-R07 | ضمان الأمان والامتثال | تطبيق معايير أمان وحماية البيانات والامتثال للمعايير المحلية والدولية | مثال: تشفير البيانات، نسخ احتياطي يومي، سجل تدقيق (Audit Trail) لجميع العمليات | سياسة أمان معتمدة وتقارير امتثال دورية | متوسطة |
