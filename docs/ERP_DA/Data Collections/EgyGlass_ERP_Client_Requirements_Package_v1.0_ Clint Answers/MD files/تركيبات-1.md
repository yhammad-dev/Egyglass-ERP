# تركيبات-1


## الرؤية والمتطلبات

| الرؤية والمتطلبات - التركيبات |  |  |  |  |  |
|---|---|---|---|---|---|
| EgyGlass - Installation | جهة الاتصال: إسلام السيد |  |  |  |  |  |
| المعرف | المتطلب | شرح المقصود | مثال توضيحي | مخرجات النظام | الأولوية |
| IMT-R01 | عرض بيانات الموقع | شاشة تعرض بيانات موقع التركيب كاملة: العنوان، التليفون، طرق الوصول | مثال: إسلام يفتح شاشة المشروع ويشوف '26 شارع النزهة، مدينة نصر، بجوار مسجد السلام' | صفحة مشروع مع بيانات الموقع | ضروري |
| IMT-R02 | عرض بيانات المسؤولين | عرض بيانات المسؤولين عن المشروع (مكتب فني، معاينات، تعاقدات) للتواصل | مثال: شاشة المشروع تظهر 'المكتب الفني: محمد فاروق 012xxxxxxx' | بطاقة فريق المشروع | ضروري |
| IMT-R03 | استلام إشعار الجاهزية | إشعار من المخازن عند اكتمال التصنيع وجاهزية المواد للتركيب | مثال: إسلام يستلم إشعار 'الأوردر ORD-2026-001 جاهز للتركيب' | إشعار جاهزية مع تفاصيل | ضروري |
| IMT-R04 | جدولة موعد التركيب | تقويم لجدولة مواعيد التركيب مع العملاء | مثال: إسلام يرتب تركيب يوم الثلاثاء 10 صباحًا في مدينة نصر مع فِرَق 1 و2 (اسم قائد الفريق) | تقويم تركيب مع إشعارات | ضروري |
| IMT-R05 | تسجيل البنود الخاصة | تسجيل بنود ما بعد التركيب: بدل كسر، خطأ تصنيع، خطأ مكتب فني، تأجيل عميل | مثال: أثناء التركيب انكسرت قطعة زجاج - إسلام يسجل 'بدل كسر - زجاج 6مم عدد 1' | نموذج بنود إضافية مع تصنيف | ضروري |
| IMT-R06 | رفع صور وتقارير | إمكانية رفع صور من الموقع وتقرير بعد انتهاء التركيبات | مثال: بعد التركيب، الفريق يرفع 5 صور للتركيب النهائي + تقرير بإتمام العمل | ألبوم صور + تقرير إلكتروني | ضروري |
| IMT-R07 | حالة المشروع الفعلية | عرض حالة المشروع في الوقت الفعلي لجميع الإدارات | مثال: كل الإدارات تشوف أن مشروع رقم 102 'تحت التركيب - تم 70%' (أو مؤجل من الموقع) | شاشة حالة مشروع مشتركة | ضروري |
