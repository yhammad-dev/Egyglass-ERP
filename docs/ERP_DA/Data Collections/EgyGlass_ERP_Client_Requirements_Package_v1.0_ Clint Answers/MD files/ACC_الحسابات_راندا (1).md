# ACC_الحسابات_راندا (1)


## الرؤية والمتطلبات

| الرؤية والمتطلبات - الحسابات |  |  |  |  |  |
|---|---|---|---|---|---|
| EgyGlass - Accounting | جهة الاتصال: راندا |  |  |  |  |  |
| المعرف | المتطلب | شرح المقصود | مثال توضيحي | مخرجات النظام | الأولوية |
| ACC-R01 | استلام الـ Quotation | استلام عرض السعر المعتمد بعد التعاقد مباشرة من النظام | مثال: بعد توقيع العقد، الـ Quotation يتجه تلقائيًا للحسابات مع جميع التفاصيل | عرض سعر إلكتروني بقسم الحسابات | ضروري |
| ACC-R02 | تسجيل الدفعة المقدمة | تسجيل الدفعة المقدمة لكل عميل وربطها بالمشروع | مثال: راندا تسجل 'دفعة مقدمة 30% من قيمة العقد = 150,000 جنيه' | سجل دفعات لكل مشروع | ضروري |
| ACC-R03 | إدارة الفواتير الإلكترونية | إصدار الفواتير إلكترونيًا مع التكامل مع منظومة ETA | مثال: راندا تصدر فاتورة ضريبية إلكترونية رقم ETA-2026-001 وترسلها للعميل | نظام فواتير إلكترونية مع تكامل ETA | ضروري |
| ACC-R04 | إدارة المستخلصات | إصدار مستخلصات للمشاريع حسب نسب الإنجاز | مثال: مشروع اكتمل 40% - راندا تصدر مستخلص رقم 1 بقيمة 200,000 جنيه | نظام مستخلصات مع تتبع نسب الإنجاز | ضروري |
| ACC-R05 | شيت عميل شامل | سجل مالي موحد لكل عميل يظهر جميع المبالغ والمدفوعات والمستحقات | مثال: شاشة عميل تظهر 'إجمالي التعاقدات: 500,000 - تم تحصيل: 350,000 - المتبقي: 150,000' | شيت عميل مالي موحد | ضروري |
