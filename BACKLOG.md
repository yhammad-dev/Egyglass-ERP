# BACKLOG — السجل الوحيد للبنود المفتوحة

> أي متطلب/فجوة/قرار **يولد في محادثة** يُقيَّد هنا **فورًا**.
> الفجوات ماضاعتش لنقص أدوار — ضاعت لأن **مفيش ليستة**.
> ملفات التحليل = المتطلبات **الأصلية**. هذا الملف = المتطلبات **المكتشَفة**.
> أي تدقيق قادم يقرأ **الاثنين معًا**.

---

## ⛔ قاعدتان لا تُخترقان

### 1. قاعدة الإغلاق
**الـ migration ليست دليل إغلاق. الكود ليس دليل إغلاق.**
بند يُغلق **فقط** بـ: البوابة **مُفعَّلة** فعليًا (لا NULL) **+** الفرع **الشغّال** مُختبَر بجلسة حقيقية.

> اكتشفنا دورين ناقصين → عملنالهم عمودين → سبناهم NULL → **الدورين فضلوا ناقصين.**

### 2. قاعدة منع الاختراع (STD-05)
**الجهل يُكتب، لا يُصمَّم.**

- أي فجوة معلومة → **تُقيَّد هنا كسؤال مفتوح**. **ممنوع** تتحول لـ `nullable` / `config` / `enum` / `flag`.
- أي مصطلح من لغة الشركة **ممنوع يدخل تصميمًا قبل تعريفه**. (كلمة "المدير التنفيذي" غير المعرَّفة أنتجت بوابة كاملة وهمية.)
- **ممنوع إعادة تفسير حقل قائم** لسد فجوة. الحقل يعني ما يعنيه في الكود — لا ما نتمناه.
- **اختبار الاختراع:** حقل/بوابة مقترَحة **ولا يمكن الإشارة إلى سطر في مستند أو قول صريح من صاحب العمل** → **اختراع. قف واسأل.**
- "القيادة عندك" = رتِّب واحسم بين بدائل عليها دليل. **ليست إذنًا بملء الفراغ.**

# ✅ قرارات محسومة — المرجع المُلزِم
### (صاحب العمل · 2026-07-12)

## السلسلة الحقيقية

```
1. INSPECTION_MANAGER: مقاسات + صور الموقع + رسم مقاسات — وبس.
   ❗ لا بوابة اعتماد. لا تحقّق على رسمة المكتب الفني.
2. مهندس المكتب الفني (TECHNICAL_OFFICE): رسم + تسعير
3. مدير المكتب الهندسي — TEC_APPROVER
      · يعتمد الرسم · يعتمد التسعير · يُصدر أمر التصنيع
4. عرض السعر → العميل → موافقة → التزام تعاقدي
5. REVIEW — قسم الجودة والمراجعة الفنية
      · مطابقة 3: طلب العميل + مقاسات/رسم المعاينات + رسم المكتب الهندسي
      · يمنع تضارب رسومات الإدارات
      · ❗ هو الذي يعتمد أمر التصنيع → بعده يتحوّل لـ PROCUREMENT
6. PROCUREMENT: فحص المخزون → تجهيز / طلب تصنيع من المصنع
7. تصنيع → استلام → تخزين → تسجيل التكاليف
8. تركيب → بنود إضافية → صور وتقرير → متابعة رضا العميل
```

## خريطة الأدوار الملزِمة

| الدور | الحقيقة |
|---|---|
| `ADMIN` (عمرو) | فواتير · خصومات · سياسات · كل الصلاحيات. **خارج سلسلة اعتماد الرسومات تمامًا.** |
| `TEC_APPROVER` | **المدير التنفيذي = مدير المكتب الهندسي.** يعتمد الرسم + التسعير + **يُصدر أمر التصنيع**. |
| `REVIEW` (محمد حسام) | **رقابة وجودة فقط.** يطابق الـ3 و**يعتمد أمر التصنيع**. **لا علاقة له بالتسعير ولا عروض الأسعار.** |
| `INSPECTION_MANAGER` (حسن بهاء) | مقاسات + صور + رسم مقاسات. **بلا بوابة اعتماد.** |
| `PROCUREMENT` (شكري) | ينفّذ الأمر **بعد** اعتماد REVIEW: مخزون · مصنع · تكاليف. |
